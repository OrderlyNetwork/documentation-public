Daily snapshots of account value plus cumulative PnL, UTC-day-aligned.

**Weight:** `5`

## Request

```json
{
  "type": "portfolio",
  "address": "0xc33fefd77012cbcf61f63e9c4745cadaadbe12f3",
  "interval": "1d",
  "start_time": 1712000000000,
  "end_time": 1715000000000,
  "limit": 90
}
```

| Field | Type | Required | Default | Notes |
| --- | --- | --- | --- | --- |
| `address` | string | Yes | — | Wallet address |
| `broker_id` | string | No | — | Optional |
| `account_id` | string | No | — | Optional |
| `interval` | string | No | `"1d"` | **Only `"1d"` supported in v1** — anything else throws `INVALID_PARAM` |
| `start_time` | int64 | No | `end_time − 90d` | ms epoch |
| `end_time` | int64 | No | `now` | ms epoch |
| `limit` | int | No | 90 | 1..365 |
| `cursor` | string | No | — | Opaque cursor |

## Response

```json
{
  "success": true,
  "data": {
    "rows": [
      {
        "account_value": "50000.5",
        "cumulative_pnl": "2500",
        "timestamp": 1715000000000
      }
    ],
    "next_cursor": null
  },
  "ts": 1779269143700
}
```

### Fields

| Field | Type | Notes |
| --- | --- | --- |
| `rows[]` | array | DESC by `timestamp` |
| `.account_value` | string | Daily collateral snapshot (`holding + frozen + upnl`) |
| `.cumulative_pnl` | string | `account_value_today − baseline_at_window_start` |
| `.timestamp` | int64 | UTC-day-aligned ms epoch |

## Notes

- Primary source is the Trino data warehouse (`user_collateral_snapshot`); falls back to MySQL `daily_user_statics` if Trino returns empty
- Throws `INVALID_PARAM` for `interval ≠ "1d"` or `limit ∉ [1, 365]`
- **v1 not yet shipped — clients should treat as optional:**
  - Per-row: `total_collateral_value`, `total_unrealized_pnl` (BE table does not split collateral / upnl)
  - Top-level: `peak_account_value`, `max_drawdown`, `max_drawdown_duration_hours` (no canonical user-level peak / drawdown algorithm yet)

For real-time collateral / upnl breakdown, use [`accountState`](/build-on-omnichain/public-info-api/account/account-state).
